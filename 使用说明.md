# Gemini Web2API 使用说明

## 项目状态

✅ 服务已启动在 `http://localhost:8081`
✅ API 端点: `http://localhost:8081/v1`
✅ API Key: `sk-gemini` (在 config.json 中配置)

## 重要提示

**这个项目需要能够访问 Google Gemini 服务 (gemini.google.com)**

如果你在中国大陆或其他无法直接访问 Google 的地区，需要配置代理。

## 配置代理（如果需要）

### 方法 1: 修改 config.json

```json
{
  "proxy": "http://127.0.0.1:7890"
}
```

### 方法 2: 命令行参数

```bash
py gemini_web2api.py --proxy http://127.0.0.1:7890
```

### 方法 3: 环境变量

```bash
set HTTPS_PROXY=http://127.0.0.1:7890
py gemini_web2api.py
```

## 使用方式

### 1. 使用 curl 测试

```bash
curl http://localhost:8081/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer sk-gemini" \
  -d "{\"model\":\"gemini-3.5-flash\",\"messages\":[{\"role\":\"user\",\"content\":\"Hello\"}]}"
```

### 2. 使用 OpenAI Python SDK

```python
import sys
import os
sys.path.insert(0, os.path.join(os.path.dirname(__file__), 'lib'))

from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:8081/v1",
    api_key="sk-gemini"
)

response = client.chat.completions.create(
    model="gemini-3.5-flash",
    messages=[{"role": "user", "content": "你好"}]
)
print(response.choices[0].message.content)
```

## 可用模型

| 模型名称 | 说明 | 输出长度 |
|---------|------|---------|
| `gemini-3.5-flash` | 快速通用模型 | ~1.2万字 |
| `gemini-3.5-flash-thinking` | 深度思考,最长输出 | ~2万字 |
| `gemini-3.5-flash-thinking-lite` | 自适应思考深度 | ~1.5万字 |
| `gemini-3.1-pro` | Pro模型(需cookie) | ~1.2万字 |
| `gemini-auto` | 自动选择模型 | 不定 |
| `gemini-flash-lite` | 轻量快速模型 | ~1万字 |

### 思考深度控制

在模型名后追加 `@think=N`:

```
gemini-3.5-flash-thinking@think=0   # 最深（默认）
gemini-3.5-flash-thinking@think=2   # 中等
gemini-3.5-flash-thinking@think=4   # 最浅
```

## 常见问题

### 1. 连接错误/超时

**原因**: 无法访问 gemini.google.com

**解决**: 配置代理（见上方"配置代理"部分）

### 2. API Key 无效

**原因**: 没有提供正确的 API key

**解决**: 在请求头中添加 `Authorization: Bearer sk-gemini`

或者修改 config.json，将 `api_keys` 设为空数组 `[]` 来禁用密钥验证。

### 3. UTF-8 解码错误

**原因**: Gemini 返回的数据编码问题

**解决**: 这通常是因为无法正常连接到 Gemini 服务，请检查代理配置。

## 停止服务

如果需要停止服务，找到进程 ID 并终止：

```bash
# 查找进程
netstat -ano | findstr :8081

# 终止进程（将 PID 替换为实际的进程ID）
taskkill /F /PID <PID>
```

## 高级配置

### 使用 Cookie（Pro 模型真实路由）

1. 访问 gemini.google.com 并登录
2. 打开开发者工具 (F12) → Application → Cookies
3. 复制 `SID`, `HSID`, `SSID`, `APISID`, `SAPISID`, `__Secure-1PSID`
4. 创建 cookie.txt:

```
SID=xxx; HSID=xxx; SSID=xxx; APISID=xxx; SAPISID=xxx; __Secure-1PSID=xxx
```

5. 修改 config.json:

```json
{
  "cookie_file": "cookie.txt"
}
```

6. 重启服务
